로직 맵

사주백과, 사주나라, 라이프운세, 사주도사 등 네 프로그램의 날짜 입력 → 산출 흐름을 동일한 캔버스에 정리하고, GitHub Pages에서 가시화한 문서입니다.

핵심 로직 파일 체크리스트

사주백과

  • data/EYCONV.db → 절기 보정 + 1st/2nd 간지 후보 (`analysis_results/EYCONV_structure.json` 참조)
  • data/GanJi_Term.DB + `.MB/.PX` → `gz_year/month/day/hour` 문자열
  • data/Select_Time.db → `IDX`, `Title`, `Data`, `korea` 기반 시생 텍스트 (`Select_Time_strings.json`)

라이프운세

  • LifeUnse/lifeunse.mdb `main(index, idx)` 기반 문구
  • `life12`, `life12Title`, `btype_couple`, `dream`, `sal`, `fourbody` 등 테이블로 주제 분기
  • 프로 버전(`lifeunsepro`, `lifeunseprojuyuk`)은 동일 구조를 복제

사주도사

  • zDosa.DB, zName.DB → 사주 인덱스 및 작명 키
  • `.dol`(life, dream1~3, hyung, osix, luck 등) 매핑으로 문구 블록 생성
  • dosa.cfg와 hangul*.fnt로 폰트/배치/오행 조건 조정

사주나라

  • 사주나라.iso 원본만 확보 → ISO 마운트 후 DB/간지 파일 추출 예정
  • 현재로선 `입력→출력` 파이프라인은 문서화 전 단계

입력 / 출력 흐름 비교

프로그램 입력 키 로직 출력
사주백과 양/음력 + 생년월일시, 성별, timezone EYCONV → GanJi_Term → Select_Time 연/월/일/시 간지, 시생 텍스트
라이프운세 사주 + 궁합/운세 옵션 `main(index, idx)` → `life12` → `dream`/`btype` 출력 케이스별 문장 블록 + 제목
사주도사 사주 + 작명/운세 선택 zDosa.DB/`zName.DB` → `.dol` 텍스트 운세·작명 텍스트 세션
사주나라 ISO → 추출 예정 DB 구조 확인 후 매핑 분석 중

통합 프레임

`docs/CORE_LOGIC_VALIDATION_SPEC.md`의 Canonical 입력/출력을 기준으로 4개 프로그램의 핵심 단계들을 정리하였습니다

단계 Canonical 필드 사주백과 라이프운세 사주도사 사주나라 (예정)
입력 `calendar_type`, `is_leap_month`, `year`, `month`, `day`, `hour`, `minute`, `timezone`, `gender` UI에서 직접 수집 MDB `index` 산출 기준 `zDosa` 인덱스 추출 ISO 내부 입력 파일 확보
중간 `solar_ymd`, `lunar_ymd`, `절기`, `ganji_index`, `hour_index` `EYCONV`/`GanJi_Term` 테이블 `life12`/`dream` 조합 `dosa.cfg` 기반 인덱스 추출 데이터로 파헤칠 영역
출력 `gz_year`, `gz_month`, `gz_day`, `gz_hour`, `시생`, `문구블럭(index+idx)` `GanJi_Term` + `Select_Time` `main.Txt` + `btype`/`dream` `*.dol` + `zName` 결과 분석 완료 후 매핑
